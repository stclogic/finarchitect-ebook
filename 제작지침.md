# bookcoupling 제작지침

이 문서는 반복 발생한 오류를 분석해 도출한 제작 규칙이다.  
신규 콘텐츠 제작·코드 수정·배포 전 반드시 이 지침에 따른다.

---

## 1. JavaScript 문자열 작성 규칙

### 규칙: 영문 텍스트에 아포스트로피(')가 포함되면 큰따옴표 사용

```js
// ❌ 오류 — 파싱 실패, 스크립트 블록 전체 실행 불가
title: 'The Wealthy's Secret Strategy',

// ✅ 올바름
title: "The Wealthy's Secret Strategy",

// ✅ 또는 백틱
title: `The Wealthy's Secret Strategy`,
```

**영향 범위:** 하나의 `<script>` 블록 안에 오류가 있으면 그 블록의 모든 코드가 실행되지 않는다.  
증상: 버튼 미작동, 목록 미표시, 언어 전환 불가.

**체크 포인트:** 영어 소유격(`'s`), 축약형(`don't`, `it's`, `you're`) 포함 문자열은 반드시 큰따옴표로 감싼다.

## 2. 다국어(KO/EN/JA) 파일 제작 순서

### 규칙: 세 언어 파일을 동시에 완성한 후 한 번에 커밋·푸시

```
✅ 올바른 순서
1. data/global.js (KO) 작성
2. data/global_en.js (EN) 작성
3. data/global_ja.js (JA) 작성
4. 세 파일 완성 확인
5. 한 번에 git add & commit & push

❌ 잘못된 순서
1. KO만 먼저 push
2. EN/JA 나중에 push  ← 불완전한 상태가 라이브에 노출됨
```

### 데이터 파일 구조 필수 항목

| 항목 | KO | EN | JA |
|---|---|---|---|
| `var VOL2_DATA = {...}` | ✅ | ✅ | ✅ |
| `lang: "ko"` | `"ko"` | `"en"` | `"ja"` |
| `volume` 번호 | 동일 | 동일 | 동일 |
| `glossary` 배열 | ✅ | ✅ | ✅ |

## 3. 캐시 버스팅 버전 파라미터 관리

### 규칙: JS/CSS 파일을 수정하면 해당 파일을 참조하는 모든 HTML의 `?v=` 파라미터를 갱신한다

```html
<!-- viewer.js 수정 시 이를 로드하는 모든 파일 갱신 -->
<script src="assets/js/viewer.js?v=20260706"></script>
```

**해당 파일 목록:**

| JS 파일 | 참조하는 HTML |
|---|---|
| `assets/js/viewer.js` | `index.html`, `index2.html`, `index3.html`, `index_en.html`, `index_ja.html` |
| `assets/js/nav.js` | 모든 페이지 (`nav.js?v=3` — 기능 변경 시 올린다) |
| `assets/js/auth.js` | 모든 페이지 |

**버전 형식:** `YYYYMMDD` (예: `v=20260706`)

## 4. Netlify 시크릿 스캔 규칙

### 규칙: 환경변수 값이 일반 영단어일 경우 `netlify.toml`에 예외 등록

현재 등록된 예외:
```toml
[build.environment]
  SECRETS_SCAN_OMIT_KEYS = "PAYPAL_MODE"
```

**발생 원인:** `PAYPAL_MODE=live` 환경변수 값 `live`가 콘텐츠 본문의 일반 단어와 일치해 false positive 발생.

**추가 예외가 필요한 경우:** 빌드 로그에서 아래 패턴 확인 후 해당 KEY를 `SECRETS_SCAN_OMIT_KEYS`에 쉼표로 추가.
```
Secret env var "KEY_NAME"'s value detected: found value at line N in file.js
```

## 5. 뷰어 파일(index*.html) 표준 구조

### 상단 네비게이션 링크 규칙

| 버튼명 | 링크 |
|---|---|
| 홈 | `landing.html` |
| 라이브러리 / eBook | `https://bookcoupling.netlify.app/library` |
| 서식 | `forms.html` |

**주의:** `eBook` 버튼은 `ebook.html`이 아닌 `library` URL로 연결한다.

### 언어 전환 구조

```js
// URL ?lang=XX 우선, 없으면 localStorage
var urlLang = new URLSearchParams(location.search).get('lang');
var lang = urlLang || localStorage.getItem('site-lang') || 'ko';
if (urlLang) localStorage.setItem('site-lang', urlLang);
var src = lang === 'en' ? 'data/xxx_en.js' : lang === 'ja' ? 'data/xxx_ja.js' : 'data/xxx.js';
document.write('<script src="' + src + '?v=YYYYMMDD"><\/script>');
```

## 6. viewer.js 블록 타입 목록

신규 섹션 작성 시 아래 타입만 사용한다. 정의되지 않은 타입은 빈 문자열로 렌더링된다.

| 타입 | 용도 |
|---|---|
| `prose` | 본문 단락 |
| `point` | 불릿 포인트 (`text` 또는 `items: []`) |
| `deep` | 심층분석 박스 (`title` + `content`) |
| `table` | 표 (`headers[]` + `rows[][]`) |
| `numlist` | 번호 목록 (`items: []`) |
| `summary` | 요약 박스 (`title` + `items: []`) |
| `book-ref` | 유료 책 인용 박스 (`text` + `url` + `btnText`) |
| `timeline` | 타임라인 |
| `cards` | 카드 그리드 |
| `case` | 케이스 스터디 |
| `story` | 연재소설 블록 |
| `divider` | 구분선 |
| `disclaimer` | 면책조항 |

### `point` 블록 두 가지 형식

```js
// 단일 텍스트
{ "type": "point", "text": "내용" }

// 배열 (여러 항목)
{ "type": "point", "items": ["항목1", "항목2", "항목3"] }
```

## 7. 신규 eBook 추가 체크리스트

새 책을 추가할 때 반드시 생성해야 하는 파일:

```
□ data/{bookid}.js          — KO 콘텐츠 데이터
□ data/{bookid}_en.js       — EN 콘텐츠 데이터
□ data/{bookid}_ja.js       — JA 콘텐츠 데이터
□ index{N}.html             — 뷰어 (paywall 구조 확인)
□ ebook{N}.html             — 랜딩 페이지
□ library.html 수정         — BOOKS 배열에 3개 언어 옵션 추가
```

**paywall 규칙:**
- 무료 책 → `index{N}.html`에서 auth IIFE 제거 (`IS_FREE_ACCESS` 미설정)
- 유료 책 → `index{N}.html`에 auth IIFE 유지, `PAYWALL_URL` 설정

## 8. 인사이트(insights/) 추가 체크리스트

```
□ insights/YYYY-MM-DD.html  — 본문 작성
□ library.html 수정         — insightList에 카드 추가 (최신 순 최상단)
□ 이전 호 insight-nav 업데이트  — "다음 호" 링크 추가
□ CTA 링크 확인             — ebook3.html / ebook2.html 연결 확인
```

## 9. 배포 전 최종 점검 목록

```
□ JS 문자열에 아포스트로피 포함 여부 확인 (영문 텍스트 전체)
□ viewer.js 수정 시 참조 HTML 전체의 ?v= 파라미터 갱신
□ 다국어 3개 파일 모두 존재 확인 (KO/EN/JA)
□ library.html 해당 책 langs 배열에 3개 언어 옵션 등록 확인
□ 상단 네비 eBook 버튼 링크 = bookcoupling.netlify.app/library
□ 새로운 영단어 env var 값이 콘텐츠에 포함될 가능성 검토
□ 브라우저 콘솔 에러 없음 확인
```
